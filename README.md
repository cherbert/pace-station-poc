# Space Station PoC — a Fabric mod for Minecraft 1.21.1

A proof-of-concept space mod. You wake up on a small space station floating in a
starfield, with a glowing **gravity generator** humming in the middle of the main
room. A **shuttle** is docked on the pad outside. Board it and fly — about 300
blocks east there's a whole spherical **planet** (grass, dirt, stone, a glowing
core) with a scattering of asteroids along the way. Land on it, walk around, fly
home.

Everything is generated by a custom dimension + chunk generator, so it works in
any new world with no setup.

## What's in the box

- `spacemod:space` dimension — a void with the station at the origin, 8 asteroids,
  and a bumpy sphere planet centred at (300, 110, 0). Locked to midnight so the
  stars are always out; no clouds, black fog.
- **Space station** — 13×13 glass-domed room, gravity generator centrepiece
  (cosmetic in this PoC), catwalk and docking pad. Built from vanilla blocks by
  the chunk generator.
- **Shuttle** — an actually pilotable ship entity with zero-g, six-degrees-of-freedom
  flight. Boat-style client-side physics (the pilot's client simulates, vanilla
  vehicle packets sync it back).
- **First join** — you're teleported to the station, your spawn point is set there,
  a shuttle is spawned on the pad, and you get a spare shuttle item.

## Building

Requirements: **JDK 21** and an internet connection (first build downloads
Minecraft, mappings and Fabric — a few minutes).

```bash
cd space-station-poc
./gradlew build          # Windows: gradlew.bat build
```

The mod jar lands in `build/libs/space-station-poc-0.1.0.jar` (ignore the
`-sources` jar).

To instead launch a dev client immediately:

```bash
./gradlew runClient
```

## Installing in Prism Launcher

1. In Prism: **Add Instance** → Minecraft **1.21.1** → under *Mod loaders* pick
   **Fabric** (latest loader) → OK.
2. Open the instance's **Mods** page → **Download mods** → search for and add
   **Fabric API** (for 1.21.1) → or drop the Fabric API jar in manually.
3. Click **Add file** (or open the instance's mods folder) and drop in
   `build/libs/space-station-poc-0.1.0.jar`.
4. Launch, create a new world (any type — survival works, creative is comfy for
   a first look). You'll appear on the station a moment after joining.

Tip: set **render distance to 16+** so the planet is visible from the station.

## Flying the shuttle

| Input | Action |
|---|---|
| Right-click shuttle | Board |
| Mouse | Aim — the ship flies where you look |
| **W** | Thrust forward (full 3D, including up/down through your crosshair) |
| **S** | Retro thrust (brake / reverse) |
| **Space** | Vertical thruster |
| **Sprint key** (Ctrl) | Boost |
| **Sneak** (Shift) | Dismount |
| F5 | Third person — recommended! |

There's drag, so let go of the keys and the ship drifts to a stop. Punch the
shuttle (not while riding) to pack it back into an item; right-click a block
with the item to redeploy it.

## Pointers for tinkering

- Planet position/size, asteroid list: constants at the top of
  `src/main/java/com/tinblue/spacemod/world/SpaceChunkGenerator.java`
- Station layout: `world/StationBuilder.java` (pure function of coordinates)
- Ship handling (thrust, drag, top speed): constants in `ShipEntity.java`
- Ship shape/texture: `client/ShipEntityModel.java` + `make_textures.py`
- Sky/fog/ambient light: `client/SpaceModClient.java` and
  `src/main/resources/data/spacemod/dimension_type/space.json`

## Known limitations / notes

- This is a PoC: the gravity generator is cosmetic, there's no oxygen system,
  and the "gravity" everywhere is ordinary Minecraft gravity — don't step off
  the station (though if you do, you'll respawn back on it).
- On a **dedicated server**, set `allow-flight=true` in `server.properties` or
  vanilla's anti-fly check may kick pilots. Singleplayer is unaffected.
- The versions in `gradle.properties` were chosen for MC 1.21.1. If Gradle
  can't resolve one (versions do get pruned/renamed), pick current 1.21.1 values
  from <https://fabricmc.net/develop> and update `gradle.properties`.
- Written for Yarn mappings on 1.21.1 specifically; other MC versions will need
  small signature fixes (e.g. `EntityType.Builder.build()` and
  `ChunkGenerator.populateNoise` changed in later versions).
